# Инструкция по синхронизации и запуску (Release 2.0)

Чтобы устранить ошибки импорта и применить актуальную архитектуру ТЕРИОН на сервере, выполните следующие шаги:

### 1. Синхронизация кода
В корневой директории проекта (`~/PARKHOMENKO_BOT`) выполните:
```bash
git fetch origin
git reset --hard origin/terion-release-2-0-final-loyalty-fix
```
*Внимание: это удалит все локальные изменения в коде, но сохранит ваш локальный `.env` (если он добавлен в .gitignore).*

### 2. Проверка конфигурации
Убедитесь, что ваш `.env` файл содержит актуальные ключи.
```bash
cat .env
```
Особенно проверьте `BOT_TOKEN` (тот, что заканчивается на `...Oiw`) и `CONTENT_BOT_TOKEN`.

### 3. Очистка кэша Python
```bash
find . -name "*.pyc" -delete
find . -name "__pycache__" -delete
```

### 4. Запуск ботов
Используйте готовые скрипты для фонового запуска:

**Запуск основного бота (Антон):**
```bash
chmod +x run_main_bot.sh
./run_main_bot.sh
```

**Запуск контент-бота (Креативщик):**
```bash
chmod +x run_bot.sh
./run_bot.sh
```

### 5. Проверка статуса
Посмотрите логи, чтобы убедиться в успешном старте:
- Основной бот: `tail -f main_bot.log`
- Контент-бот: `tail -f bot.log`

Если вы видите «Бот успешно запущен...» или «Креативщик запущен...», система готова к работе. Попробуйте отправить `/start` в обоих ботах.
