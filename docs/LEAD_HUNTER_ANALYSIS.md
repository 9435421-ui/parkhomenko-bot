## 🎯 2. ОПРЕДЕЛЕНИЕ ЛИДА

### Критерии определения лида

**Метод:** `ScoutParser.detect_lead()` (Intent v3.0)

#### HOT_TRIGGERS (критические фразы — лид сразу):
- `"предписание МЖИ"`, `"узаконить"`, `"МЖИ"`, `"предписание"`, `"инспектор МЖИ"`, `"пришла МЖИ"`, `"штраф БТИ"`, `"штраф за перепланировку"`, `"блокировка сделки"`, `"суд по перепланировке"`

#### Смягченные фильтры (лид = [Тех. термин] + [Вопрос ИЛИ Коммерческий маркер]):

**Технические термины:**
- `"перепланиров"`, `"согласовани"`, `"узакони"`, `"МЖИ"`, `"акт скрытых работ"`, `"проект СРО"`, `"мокрая зона"`, `"снос стен"`, `"объединен"`, `"нежилое помещение"`, и др.

**Паттерны вопроса:**
- `"кто согласовывал"`, `"как согласовать"`, `"подскажите"`, `"посоветуйте"`, `"можно ли"`, `"где"`, `"?"` (в конце)

**Коммерческие маркеры:**
- `"стоимость"`, `"сколько стоит"`, `"сроки"`, `"кто делал"`, `"к кому обратиться"`, `"нужен проект"`, `"заказать проект"`

**Отсечение мусора:**
- `"продам"`, `"аренда"`, `"услуги сантехника"`, `"ремонт под ключ"`, `"вызов сантехника"`, `"мастер на час"`

**Минимальные требования:**
- Минимум 5 слов в сообщении
- Не только ссылки без текста

### Поля лида в БД (`spy_leads`)

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER PRIMARY KEY | Уникальный ID лида |
| `source_type` | TEXT | `"telegram"` или `"vk"` |
| `source_name` | TEXT | Название канала/группы (например, "ЖК Зиларт") |
| `author_id` | TEXT | ID автора сообщения (Telegram user_id или VK user_id) |
| `username` | TEXT | Username автора (если есть) |
| `profile_url` | TEXT | Ссылка на профиль (`tg://user?id=...` или `https://vk.com/id...`) |
| `text` | TEXT | Текст сообщения/поста (до 2000 символов) |
| `url` | TEXT | Ссылка на пост/комментарий |
| `pain_stage` | TEXT | Стадия боли: `"ST-1"` (Интерес), `"ST-2"` (Планирование), `"ST-3"` (Актив), `"ST-4"` (Критично) |
| `priority_score` | INTEGER | Приоритет лида (1-10) |
| `sent_to_hot_leads` | INTEGER | Флаг отправки в топик "Горячие лиды" (0/1) |
| `contacted_at` | TIMESTAMP | Время первого контакта (для модуля "Продавец") |
| `created_at` | TIMESTAMP | Время создания записи |

### Пороги отсечения

| Критерий | Порог | Действие |
|---------|-------|----------|
| **Горячий лид** | `HOT_TRIGGERS` ИЛИ `pain_stage` в (`ST-1`, `ST-2`, `ST-3`, `ST-4`) ИЛИ `priority_score >= 3` ИЛИ `hotness >= 4` | Немедленная отправка в топик "Горячие лиды" (`THREAD_ID_HOT_LEADS = 811`) |
| **Обычный лид** | `priority_score < 3` И нет `HOT_TRIGGERS` И `pain_stage` не `ST-3`/`ST-4` | Сохранение для сводки (отправка 3 раза в день: 10:00, 14:00, 19:00 МСК) |
| **ST-3/ST-4 в нерабочее время** | `pain_stage` в (`ST-3`, `ST-4`) И время вне 09:00-20:00 МСК | Отложенная отправка до рабочего времени |
| **Гео-фильтрация** | Не Москва/МО (маркеры других регионов) | Отсеивание на этапе анализа |

### Ключевые слова и фильтры

**Источники ключевых слов:**
1. **Код:** `ScoutParser.KEYWORDS` (70+ слов)
2. **БД:** Таблица `spy_keywords` (управление через админ-панель)
3. **.env:** `SCOUT_TG_KEYWORDS`, `SCOUT_VK_KEYWORDS` (через запятую)

**Приоритетные ЖК (автоматическое повышение приоритета):**
- Зиларт, Символ, Сердце Столицы, Династия, Пресня Сити, Некрасовка, Люблинский парк, Фили Град, Волжский парк, Саларьево, Митино, Сколково, Братеево

## 🔄 3. ЦИКЛ РАБОТЫ ОХОТНИКА

### Главная функция: `LeadHunter.hunt()`

**Расположение:** `services/lead_hunter/hunter.py:609`

**Шаги выполнения:**

1. **Очистка кеша парсера** (строки 615-621)
   - Сброс `last_scan_report`, `last_scan_chats_list`
   - Установка `last_scan_at = datetime.now()`

2. **Подключение к БД** (строка 624)
   - Проверка и переподключение при необходимости

3. **Парсинг источников** (строки 626-628)
   - `tg_posts = await self.parser.parse_telegram(db=main_db)` — парсинг Telegram
   - `vk_posts = await self.parser.parse_vk()` — парсинг VK
   - `all_posts = tg_posts + vk_posts`

4. **Discovery (если лидов не найдено)** (строки 631-698)
   - Поиск новых Telegram каналов через `discovery.find_new_sources()`
   - Поиск новых VK групп через `discovery.scout_vk_resources()`
   - Добавление найденных источников в `target_resources` со статусом `active`

5. **Приоритизация чатов** (строки 704-711)
   - Приоритетные чаты (ЖК Династия, Зиларт) перемещаются в начало списка
   - Настройка через `.env`: `SPY_PREFERRED_CHATS="Династия,Зиларт"`

6. **Обработка каждого поста** (строки 726-1069)
   - **Анти-дубль:** проверка `_seen_post_keys` (один пост не обрабатывается дважды)
   - **Быстрая оценка:** `analyzer.analyze_post()` → `priority_score`, `pain_stage`
   - **Гео-фильтрация:** отсеивание лидов не из Москвы/МО
   - **Глубокий анализ:** `_analyze_intent()` через Yandex GPT агент
   - **Сохранение в HunterDatabase:** если `is_lead == True` → сохранение в `potential_leads.db`
   - **Уведомление Юлии:** отправка карточки лида в личку (`JULIA_USER_ID`)
   - **Обработка через AI Жюля:** `hunter_standalone.hunt()` → анализ горячести
   - **Классификация лидов:**
     - **Горячие:** `HOT_TRIGGERS` ИЛИ `ST-1`/`ST-2` → немедленная отправка в топик "Горячие лиды"
     - **ST-3/ST-4:** отправка только в рабочее время (09:00-20:00 МСК)
     - **Обычные:** сохранение для сводки

7. **Отправка отчёта** (строки 1103-1124)
   - Отчёт о сканировании в топик "Логи" (`THREAD_ID_LOGS = 88`)
   - Формат: список каналов/групп, количество просмотренных сообщений, количество найденных лидов

8. **Отправка файла со списком лидов** (строки 1126-1128)
   - Текстовый файл со всеми лидами (источник, превью текста, ссылка) в топик "Логи"

### Периодичность сканирования

| Задача | Периодичность | Функция | Файл |
|--------|---------------|---------|------|
| **Основная охота** | Каждые 30 минут | `hunter.hunt()` | `main.py:178` |
| **Сводка обычных лидов** | 3 раза в день: 10:00, 14:00, 19:00 МСК | `hunter.send_regular_leads_summary()` | `main.py:201-221` |

**Уровни:**

## 📍 4. МЕСТА, ГДЕ "ШПИОН" РАБОТАЕТ

### Чтение сообщений/постов

**Telegram (`scout_parser.py:498`):**
- **Источники:** 
  - Из БД: `get_active_targets_for_scout()` (таблица `target_resources`, статус `active`, платформа `telegram`)
  - Из конфига: `SCOUT_TG_CHANNELS` (через `.env`)
- **Метод:** `Telethon.iter_messages()` — итерация по сообщениям канала
- **Инкрементальный поиск:** Используется `last_post_id` из БД для каждого ресурса (сканирование только новых сообщений)
- **Прогревочный скан:** Для новых каналов (`last_post_id == 0`) — последние 20 сообщений
- **Discussion Groups:** Автоматический переход к связанному чату для комментариев
- **Фильтрация:** Только сообщения от `User`, не от `Channel` (игнорируются посты от самого канала)

**VK (`scout_parser.py:1075`):**
- **Источники:**
  - Из БД: `get_active_targets_for_scout()` (статус `active`, платформа `vk`)
  - Из конфига: `SCOUT_VK_GROUPS` (через `.env`)
- **Метод:** VK API `wall.get` (посты на стене) + `wall.getComments` (комментарии к постам)
- **Лимиты:** 
  - По умолчанию: последние 50 постов на стене
  - Комментарии: последние 10 постов × 30 комментариев (настраивается через `.env`)

### Принятие решения "лид / не лид"

**Метод:** `ScoutParser.detect_lead()` (строка 363)

**Логика:**
1. **Проверка HOT_TRIGGERS** → если найдено → **ГОРЯЧИЙ ЛИД** (возврат `True`)
2. **Отсечение мусора** → если найдены `JUNK_PHRASES` → возврат `False`
3. **Проверка фильтров:**
   - Есть технический термин (`_has_technical_term()`)
   - И (есть вопрос (`_has_question()`) ИЛИ есть коммерческий маркер (`_has_commercial_marker()`))
4. **Дополнительная проверка:** совпадение с `LEAD_TRIGGERS` или ключевыми словами

**AI-анализ:** `LeadAnalyzer.analyze_post()` (строка 160)
- Гео-фильтрация (Москва/МО)
- Определение `priority_score` (1-10) и `pain_stage` (ST-1…ST-4)
- Проверка приоритетных ЖК
- Использование Router AI / Yandex GPT для глубокого анализа

### Отправка результата

**Горячие лиды** → Топик "Горячие лиды" (`THREAD_ID_HOT_LEADS = 811`):
- Метод: `_send_lead_card_to_group()` (строка 425)
- Формат: HTML-карточка с кнопками:
  - `👤 Профиль` (если есть `profile_url`)
  - `🔗 Пост` (ссылка на пост)
  - `✍️ На эту тему пост` (создание контента)
  - `🛠 Ответить экспертно` (генерация ответа от Антона)
  - `✅ Взять в работу` (контакт Юлии)

**Обычные лиды** → Топик "Логи" (`THREAD_ID_LOGS = 88`):
- Метод: `send_regular_leads_summary()` (строка 1132)
- Формат: Сводка до 20 лидов с превью текста, приоритетом, стадией боли, ссылкой на пост
- Расписание: 10:00, 14:00, 19:00 МСК

**Отчёт о сканировании** → Топик "Логи":
- Метод: `parser.get_last_scan_report()` (строка 1106)
- Формат: Список каналов/групп, количество просмотренных сообщений, количество найденных лидов

**Файл со списком лидов** → Топик "Логи":
- Метод: `_send_raw_leads_file_to_group()` (строка 508)
- Формат: Текстовый файл `.txt` с источником, превью текста, ссылкой для каждого лида

**Уведомление Юлии** → Личное сообщение (`JULIA_USER_ID`):
- Метод: `_send_lead_notify_to_admin()` (строка 539) — для каждого лида (если включено в настройках)
- Метод: Прямая отправка карточки лида (строки 777-891) — для новых лидов с проектом ответа

**Личное сообщение пользователю** → Telegram DM:
- Метод: `_send_dm_to_user()` (строка 359)
- Условие: Лид найден в открытом чате И есть `author_id`
- Формат: Приветствие от Антона + ссылка на квиз

## 🔍 7. КАК СЕЙЧАС РАБОТАЕТ ШПИОН

**Шпион** — это автоматизированная система поиска потенциальных клиентов в Telegram каналах и VK группах, которая работает следующим образом:

1. **Планировщик задач** (`main.py`) запускает основную функцию `hunter.hunt()` каждые 30 минут через APScheduler.

2. **Парсинг источников:** `ScoutParser` получает список активных каналов/групп из БД (`target_resources`, статус `active`) и сканирует их через Telethon (Telegram) и VK API. Для каждого ресурса используется инкрементальный поиск через `last_post_id` — сканируются только новые сообщения после последнего обработанного поста.

3. **Фильтрация сообщений:** Для каждого сообщения проверяется, является ли оно лидом через метод `detect_lead()`. Система проверяет наличие критических фраз (`HOT_TRIGGERS`), технических терминов, вопросов или коммерческих маркеров, отсеивая мусор (рекламу, объявления без запросов).

4. **AI-анализ:** Найденные посты анализируются через `LeadAnalyzer`, который определяет `priority_score` (1-10) и `pain_stage` (ST-1…ST-4), применяет гео-фильтрацию (только Москва/МО) и проверяет приоритетные ЖК для автоматического повышения приоритета.

5. **Классификация и отправка:** Лиды классифицируются на горячие (немедленная отправка в топик "Горячие лиды") и обычные (сохранение для сводки). Горячие лиды отправляются сразу, ST-3/ST-4 — только в рабочее время (09:00-20:00 МСК), обычные лиды собираются в сводку, которая отправляется 3 раза в день (10:00, 14:00, 19:00 МСК).

6. **Discovery:** Если лидов не найдено, система автоматически ищет новые источники через `Discovery`, используя глобальный поиск Telegram и VK API по ключевым словам, и добавляет найденные каналы/группы в БД со статусом `active`.

7. **Отчёты:** После каждого скана система отправляет отчёт в топик "Логи" с информацией о том, где был шпион, сколько сообщений просмотрено и сколько лидов найдено, а также текстовый файл со списком всех лидов.

## 🎯 10. КРАТКОЕ ОПИСАНИЕ РАБОТЫ ШПИОНА

**Шпион** работает в автоматическом режиме, сканируя Telegram каналы и VK группы каждые 30 минут. Он получает список активных источников из базы данных (`target_resources`), использует инкрементальный поиск через `last_post_id` для сканирования только новых сообщений и проверяет каждое сообщение на наличие ключевых слов через метод `detect_lead()`. Найденные посты анализируются через AI (`LeadAnalyzer`), который определяет приоритет (1-10) и стадию боли клиента (ST-1…ST-4), применяет гео-фильтрацию (только Москва/МО) и проверяет приоритетные ЖК. Горячие лиды (HOT_TRIGGERS, ST-1/ST-2, priority_score >= 3) отправляются немедленно в топик "Горячие лиды" с карточкой и кнопками действий, обычные лиды сохраняются для сводки, которая отправляется 3 раза в день (10:00, 14:00, 19:00 МСК) в топик "Логи". После каждого скана система отправляет отчёт о том, где был шпион и сколько лидов найдено, а также текстовый файл со списком всех лидов. Если лидов не найдено, система автоматически ищет новые источники через Discovery и добавляет их в базу данных для дальнейшего мониторинга.
