# Снайперский мониторинг жилых ЖК

Шпион перенастроен на поиск «горячих» проблем в обжитых домах: локальные чаты ЖК вместо общих чатов по недвижимости.

## Приоритетные ЖК для мониторинга

| ЖК | Описание |
|----|----------|
| ЖК «Сердце Столицы» | Сложные инженерные решения |
| ЖК «Символ» | Много квартир без перегородок |
| ЖК «Зиларт» | Активные ремонты и строгая УК |
| ЖК «Пресня Сити» | Вопросы по апартаментам |
| Сити (Башни) | Коммерция и апартаменты |

## Настройка чатов в .env

ID чатов Telegram задаются через переменные (без них каналы с пустым `id` пропускаются):

- `SCOUT_TG_CHANNEL_1_ID` — ID или username чата ЖК «Сердце Столицы»
- `SCOUT_TG_CHANNEL_1_NAME` — `ЖК «Сердце Столицы»`
- `SCOUT_TG_CHANNEL_1_GEO` — `Москва`
- Аналогично 2–5 для остальных ЖК.

Пример: `SCOUT_TG_CHANNEL_1_ID=-1001234567890` или username чата.

## Интеллектуальный фильтр (Intent)

Лидом считается **только** сообщение, в котором есть **и вопрос, и технический термин**.

- **Пример мусора (игнорируем):** «Посоветуйте рабочих» — нет технического термина.
- **Пример лида (берём):** «Соседи, кто согласовывал снос подоконного блока в нашем корпусе?» — вопрос + термин.

Технические термины: перепланировка, согласование, МЖИ, предписание МЖИ, штраф за перепланировку, акт скрытых работ, проект СРО, согласие соседей, мокрая зона, снос стен/подоконного блока и др. (см. `TECHNICAL_TERMS` и `KEYWORDS` в `scout_parser.py`).

## Расширение триггеров (боли жильцов)

Добавлены ключи: **предписание МЖИ**, **штраф за перепланировку**, **акт скрытых работ**, **проект СРО**, **согласие соседей**, **мокрая зона**.

## Гео-привязка

Если в сообщении есть номер корпуса или название ЖК, это выносится в **заголовок карточки лида** (например: «ЖК «Зиларт», корп. 5»). Метод: `ScoutParser.extract_geo_header(text, source_name)`.

## Результат

В рабочую группу попадают не «180 случайных сообщений», а **5–10 карточек в день** — сообщения, где человек уже ищет эксперта или столкнулся с проблемой.
