**Дата:** 19.02.2026

---

## 🎯 Общая стратегия

**Переход от привязки к конкретным ЖК к глобальному поиску с гео-фильтрацией:**

- ❌ **Старая стратегия:** Мониторинг фиксированного списка ЖК-специфичных каналов
- ✅ **Новая стратегия:** Глобальный поиск каналов по ключевым словам + фильтрация только по гео (Москва/МО)

## ✅ Реализованные модули

### 1. 🔍 Discovery Engine (Автопоиск каналов)

**Файл:** `services/lead_hunter/discovery.py`

#### Расширенные ключевые слова

**Общие запросы** (комбинируются с районами):
- "перепланировка москва"
- "согласование перепланировки"
- "БТИ москва чат"
- "дизайн интерьера чат"
- "ремонт квартир мск"

**Географические запросы** (чаты жильцов):
- "ЖК Москва чат"
- "соседи ЖК"
- "новостройки москва обсуждение"
- "чат жильцов"

**Районы Москвы для комбинирования:**
- ЮВАО, ЮАО, ЮЗАО, СВАО, САО, СЗАО, ВАО, ЦАО, ЗАО, НАО, ТАО

**Результат:** Генерируется **100+ комбинаций** ключевых слов для поиска сотен чатов

#### Умная логика SCOUT_KEYWORDS

- Если в `.env` задано `SCOUT_KEYWORDS` с **менее 5 словами** → игнорируется, используется расширенный список
- Если задано **5+ слов** → используется из env
- Добавлено логирование выбора источника ключевых слов

**Пример:** Если в `.env` задано `SCOUT_KEYWORDS="чат"`, система автоматически использует расширенный список вместо одного слова.

### 2. 💬 Discussion Groups (Парсинг комментариев)

**Файл:** `services/scout_parser.py`

#### Реализовано:

✅ **Автоматическое обнаружение Discussion Groups:**
- Использует `GetFullChannelRequest` для получения `discussion_group_id`
- Автоматически находит связанные чаты для комментариев

✅ **Принудительный парсинг комментариев:**
- Основной канал **игнорируется** (только посты от админов)
- Фокус **только на комментариях** от пользователей в Discussion Group
- Комментарии помечаются как `is_comment=True` с указанием оригинального канала

✅ **Фильтрация по типу отправителя:**
- Парсятся **только сообщения от User**, не от Channel
- Пропускаются боты и другие типы отправителей

**Логирование:**
```
💬 Discovery: у канала [название] найден Discussion Group (ID: ...)
💬 Парсинг комментариев из Discussion Group канала [название]...
💬 Найден лид в комментариях: [текст]...
```

### 3. 🚫 Фильтры по типу отправителя

**Файл:** `services/scout_parser.py`

#### Игнорирование постов от каналов (админов):

✅ **Для каналов:**
- Основной поток сообщений **полностью пропускается**
- Парсятся **только комментарии** из Discussion Group

✅ **Для групповых чатов:**
- Проверка типа отправителя через `isinstance(message.sender, User)`
- Пропуск сообщений от `Channel` типа
- Дополнительная проверка: `if sender_id == peer_id.channel_id: continue`

**Результат:** Шпион обрабатывает **только сообщения от реальных пользователей**, не от админов каналов.

### 4. 📏 Фильтр по длине сообщений

**Файл:** `services/lead_hunter/analyzer.py`

#### Реализовано:

✅ **Отсеивание статей:**
- Сообщения **длиннее 500 символов** → пропускаются (это статьи, не лиды)
- Добавлено логирование: `🚫 Фильтр длины: сообщение слишком длинное (X символов)`

**Логика:**
```python
if len(text) > 500:
    return {"is_lead": False, "length_filtered": True}
```

**Результат:** Шпион фокусируется на **коротких вопросах и запросах**, а не на длинных постах-статьях.

**Файл:** `services/lead_hunter/analyzer.py`

- "посоветуйте", "подскажите", "подскажите пожалуйста"
- "сколько стоит", "сколько будет стоить", "какая цена"
- "можно ли", "можно", "можно?"

**Результат:** Шпион обрабатывает **только вопросы и запросы помощи**, игнорируя утверждения и информационные посты.

### 7. ⭐ Приоритетные ЖК

**Файл:** `services/lead_hunter/analyzer.py`

#### Реализовано:

✅ **Список приоритетных ЖК (13 ЖК):**
- Зиларт, Символ, Сердце Столицы, Династия, Пресня Сити
- Некрасовка, Люблинский парк, Фили Град, Волжский парк
- Саларьево, Митино, Сколково, Братеево

✅ **Автоматическая пометка:**
- Если в тексте упоминается приоритетный ЖК → лид помечается как `⭐ ПРИОРИТЕТНЫЙ`
- Добавляется поле `is_priority_zhk: True` и `zhk_name`

✅ **Быстрый триггер:**
- Если приоритетный ЖК + проблемный контекст → **немедленно ST-4** (самая высокая стадия боли)

**Результат:** Лиды из приоритетных ЖК получают **максимальный приоритет** и обрабатываются в первую очередь.

- Районы: "ювао", "юао", "юзао", "свао", "сао", "сзао", "вао", "цао", "зао", "нао", "тао"

### 9. 💬 Отправка ЛС от Антона

**Файл:** `services/lead_hunter/hunter.py`

#### Реализовано:

✅ **Автоматическая отправка ЛС при обнаружении лида:**
- Если лид найден в открытом чате и есть `author_id` → отправляется ЛС
- Сообщение от Антона с представлением как ИИ-ассистента Юлии Пархоменко
- Цитата вопроса пользователя (первые 200 символов)
- Ссылка на пост для контекста
- Приглашение пройти квиз

✅ **Обработка ошибок:**
- Игнорируются ошибки "bot blocked by user" и "chat not found"
- Логирование успешных отправок и ошибок

**Формат сообщения:**
```
Здравствуйте! Я Антон, ИИ-ассистент эксперта Юлии Пархоменко (компания GEORIS).

Увидел ваш вопрос по поводу перепланировки:
«[цитата вопроса]»

Могу помочь с согласованием перепланировки в Москве.
Для начала ответьте на несколько вопросов в нашем квизе:
https://t.me/Parkhovenko_i_kompaniya_bot?start=quiz

🔗 Ваш пост: [ссылка]
```

**Результат:** При обнаружении лида Антон **автоматически пишет пользователю в ЛС**, повышая конверсию.

### 10. 🔄 Активация существующих каналов

**Файл:** `services/lead_hunter/hunter.py`

#### Реализовано:

✅ **Умная логика добавления каналов:**
- Если канал уже есть в БД со статусом `"archived"` или `"pending"` → **активируется**
- Если канал новый → добавляется со статусом `"active"`
- Если канал уже активен → пропускается

✅ **Логирование:**
- `✅ Discovery: добавлен канал [название]`
- `🔄 Discovery: активирован канал [название] (был: archived/pending)`
- `📊 Discovery: добавлено X новых каналов в БД (статус: active)`

**Результат:** Discovery **автоматически активирует** ранее заархивированные каналы, если они снова найдены.

## 📋 Порядок применения фильтров

1. **Гео-фильтрация** → отсеивание не из Москвы/МО
2. **Фильтр по длине** → отсеивание статей (> 500 символов)
3. **Фильтр по типу контента** → только вопросы и запросы помощи
4. **Фильтр по типу отправителя** → только от User, не от Channel
5. **Trigger words** → автоматическое определение горячих лидов
6. **Приоритетные ЖК** → пометка приоритетных лидов
7. **Глубокий AI-анализ** → для остальных лидов

## 🗄️ База данных

### Таблица `target_resources`:

- **Статусы:** `pending`, `active`, `archived`
- **Поля:** `link`, `title`, `notes`, `status`, `participants_count`, `geo_tag`, `is_high_priority`
- **Платформы:** `telegram`, `vk`

### Методы работы с целями:

- `add_target_resource()` — добавление/обновление цели

- `set_target_status()` — изменение статуса (активация/архивация)
- `get_target_resource_by_link()` — поиск по ссылке

## 🎯 Итоговый результат

### Что умеет шпион:

1. ✅ **Автоматически находит** сотни каналов по ключевым словам с комбинациями районов
2. ✅ **Парсит комментарии** из Discussion Groups, игнорируя посты от админов
3. ✅ **Фильтрует лиды** по гео (только Москва/МО), длине (только короткие вопросы), типу контента (только вопросы)
4. ✅ **Определяет горячие лиды** по trigger words и приоритетным ЖК
5. ✅ **Автоматически пишет в ЛС** пользователям при обнаружении лида
6. ✅ **Активирует** ранее заархивированные каналы при повторном обнаружении

### Метрики эффективности:

- **Охват:** Сотни каналов вместо фиксированного списка
- **Точность:** Многоуровневая фильтрация (гео + длина + тип контента + тип отправителя)
- **Скорость:** Быстрые триггеры для горячих лидов (без ожидания AI)
- **Конверсия:** Автоматическая отправка ЛС от Антона
