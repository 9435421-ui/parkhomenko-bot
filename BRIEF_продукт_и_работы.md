# БРИФ: Продукт и Работы

## 1. Общее описание проекта

**ТЕРИОН** — система автоматизации поиска и обработки лидов по перепланировкам недвижимости. Состоит из нескольких взаимосвязанных компонентов:

- **ИИ-консультант Антон** — автоматические консультации через Telegram-бота
- **Scout Parser** — мониторинг и парсинг лидов из Telegram и VK
- **LeadHunter** — анализ и генерация ответов на лиды
- **Content Factory** — автоматическая генерация и публикация контента
- **AutoPoster** — автоматическая публикация постов в каналы и MAX.ru
- **Watchdog** — система самовосстановления и мониторинга процессов
- **SessionManager** — управление авторизацией Telethon сессий

---

## 2. Ключевые файлы

### Главный входной файл
- **`main.py`** — основной файл запуска системы. Инициализирует базу данных, запускает планировщик задач (AsyncIOScheduler), управляет двумя ботами (main_bot и content_bot) с раздельными Dispatchers.
- **`watchdog.py`** — система самовосстановления. Автоматически перезапускает main_bot и lead_hunter при падении, записывает ошибки в БД.

### Основные модули

#### База данных
- **`database/db.py`** — работа с SQLite базой данных (parkhomenko_bot.db). Автоматические миграции при старте.

#### Сервисы поиска лидов
- **`services/scout_parser.py`** — парсинг сообщений из Telegram и VK с фильтрацией по ключевым словам и анти-спам проверками
- **`services/lead_hunter/discovery.py`** — глобальный поиск новых чатов/групп через Telegram SearchGlobalRequest
- **`services/lead_hunter/hunter.py`** — анализ лидов и генерация ответов с использованием ИИ, включая еженедельный Insight of the Week

#### Контент и публикация
- **`auto_poster.py`** — автоматическая публикация постов из контент-плана в Telegram-каналы
- **`agents/content_agent.py`** — генерация текстов постов с использованием Yandex GPT
- **`agents/image_agent.py`** — генерация изображений для постов
- **`services/publisher.py`** — публикация в Telegram, VK и MAX.ru с форматированием

#### Авторизация и сессии
- **`session_manager.py`** — управление сессией Telethon для парсинга чатов. Запрашивает код подтверждения при первом запуске.

#### Обработчики бота
- **`handlers/start.py`** — обработка команды /start и квиза
- **`handlers/dialog.py`** — диалоговый режим консультаций
- **`handlers/admin.py`** — административные команды
- **`handlers/content.py`** — управление контентом

#### Конфигурация
- **`config.py`** — загрузка переменных окружения из `.env`
- **`.env.example`** — шаблон переменных окружения

## 3. LeadHunter + ScoutParser

### ScoutParser — Мониторинг лидов

**Функционал:**
- Парсинг сообщений из Telegram-групп и VK-групп
- Фильтрация по ключевым словам (`KEYWORDS`, `TECHNICAL_TERMS`)
- Приоритизация по приоритетным ЖК (`PRIORITY_ZHK_NAMES`)
- Инкрементальный парсинг через `last_post_id`
- Гео-тегирование лидов

**Внедрение фильтров "Анти-Спам/Анти-Бот" (6 уровней проверки):**

1. **Исключение каналов** — если сообщение пришло от имени канала (sender_type — канал, а не пользователь), игнорируется
2. **Черный список фраз (реклама)** — список `AD_STOP_WORDS`: "подпишись", "наш канал", "выплата", "канал о недвижимости", "заходи в группу" и т.д.
3. **Детектор рекламных ссылок** — если в одном сообщении более 2-х ссылок или есть ссылка на другой Telegram-канал — в модерацию не отправляется
4. **Фокус на запрос** — остаются только сообщения, где есть вопросительные знаки или слова-маркеры боли (из списка `QUESTION_PATTERNS` и `HOT_TRIGGERS`)
5. **Проверка истории** — если мы уже писали этому пользователю в последние 48 часов, не отправляется повторно
6. **STOP_KEYWORDS** — базовые стоп-слова для исключения нерелевантных тем

### Режим ручной модерации через админку

**При обнаружении лида:**
- Бот формирует расширенную карточку в админ-канал (`LEADS_GROUP_CHAT_ID`, топик `THREAD_ID_HOT_LEADS`)
- В карточке отображаются:
  - **Платформа**: TG/VK
  - **Локация**: Geo Header (например: "ЖК Зиларт, корп. 5")
  - **Приоритет**: Метка "🔥 Высокий", если это приоритетный ЖК
  - **Текст сообщения**: обрезанный до 300 символов
  - **Ссылка на пост**: если доступна

**Интерактивные кнопки:**
- **✅ Одобрить (Антон пишет)** — использует LeadHunter для генерации экспертного ответа, показывает на финальное подтверждение
- **✏️ Редактировать** — ручной ввод ответа
- **🗑 Пропустить** — пропустить лид

**Важно:** Автоматическая отправка ответов лидам **отключена**. Все ответы отправляются только после одобрения через админ-панель.

### LeadHunter — Анализ и генерация ответов

**Функционал:**
- Анализ интента лида через ИИ
- Определение стадии боли (ST-1 до ST-4)
- Генерация персонализированных ответов
- Подбор релевантных кейсов из портфолио
- **Insight of the Week**: еженедельный анализ топовых лидов ST-3/ST-4 для генерации контента в MAX.ru (воскресенье 18:00)
- **Резонанс-фильтр**: автоматическое уведомление админам при постах в VK с 5+ комментариями для потенциального контента

**3-х ступенчатый Fallback для генерации ответов:**

1. **Yandex GPT (основной ключ)** — `YANDEX_API_KEY` из `.env`
2. **Yandex GPT (резервный ключ)** — `YANDEX_API_KEY_BACKUP` из `.env` (если основной не сработал)
3. **Router AI** — `OPENROUTER_API_KEY` из `.env` (если оба ключа Яндекса не сработали)
4. **Статические шаблоны** — последний резерв по стадии боли

**3-х ступенчатый Fallback для анализа интента:**

Аналогичный fallback реализован в методе `_analyze_intent` для обеспечения надежности анализа лидов даже при сбое основного API-ключа.

## 4. Агент-Продавец (LeadHunter)

**Текущая реализация:**

- **Анализ интента** — определение намерений клиента через ИИ
- **Определение стадии боли** — классификация по шкале ST-1 (интерес) до ST-4 (критическая ситуация)
- **Генерация ответов** — персонализированные ответы с учетом геолокации и интента
- **Подбор кейсов** — сопоставление с похожими кейсами из портфолио (заглушка, будет реализовано позже)

**3-х ступенчатый Fallback:**
- Yandex GPT (основной ключ) → Yandex GPT (резервный ключ) → Router AI → Статические шаблоны

**Режим модерации:**
- Все ответы проходят через админ-панель перед отправкой
- Кнопки "Одобрить", "Редактировать", "Пропустить" для каждого лида

## 5. Content Factory + AutoPoster

### Content Agent

**Функционал:**
- Генерация текстов постов по типам: `news`, `fact`, `case`, `offer`, `seasonal`
- Использование Yandex GPT для генерации
- Асинхронные HTTP-запросы через `aiohttp`

### AutoPoster

**Функционал:**
- Автоматическая публикация постов из таблицы `content_plan` (статус `approved`, `publish_date <= сейчас`)
- Загрузка изображений по `image_url` (поддержка HTTP URL и Telegram file_id)
- HTML-форматирование постов
- Публикация в канал `CONTENT_CHANNEL_ID`
### Publisher (MAX.ru интеграция)

**Функционал:**
- Публикация в Telegram, VK и MAX.ru
- Форматирование постов для MAX.ru с брендингом "Антон, ИИ-ассистент компании TERION"
- Автоматическая генерация инсайтов из лидов ST-3/ST-4
**Интеграция в main.py:**
- Задача планировщика `check_and_publish_scheduled_posts()` запускается по расписанию
- Логирование: "Постов к публикации не найдено" если нет постов для публикации

## 7. База данных

### Основные таблицы

- **`sources`** — источники для мониторинга (Telegram/VK группы)
  - Поля: `id`, `type`, `source_id`, `name`, `category`, `keywords`, `is_active`, `last_scanned`, `updated_at`
  - Автоматическая миграция при старте: добавление `updated_at` если отсутствует

- **`leads`** — обнаруженные лиды
  - Поля: `id`, `source_id`, `text`, `author_id`, `author_name`, `url`, `published_at`, `status`, `platform`, `geo_tag`, `participants_count`, `is_high_priority`, `last_lead_at`, `contacted_at`

- **`content_plan`** — план публикации контента
  - Поля: `id`, `title`, `body`, `cta`, `status`, `publish_date`, `image_prompt`, `image_url`, `type`, `theme`

- **`unified_leads`** — унифицированные лиды из всех источников

### Миграции

- Автоматические миграции при старте бота (например, добавление `updated_at` в `sources`)
- Файлы миграций в папке `migrations/`

## 8. Планировщик задач (main.py)

**Задачи по расписанию:**

1. **ScoutParser.hunt()** — периодический парсинг источников (Telegram/VK)
2. **LeadHunter.hunt()** — анализ новых лидов и отправка карточек в модерацию
3. **LeadHunter.generate_weekly_insight()** — еженедельный анализ топовых лидов ST-3/ST-4 (воскресенье 18:00)
4. **check_and_publish_scheduled_posts()** — публикация постов из контент-плана
5. **competitor_spy.scan_geo_chats()** — мониторинг конкурентных чатов (каждые 5 минут)
6. **creative_agent.scout_topics()** — поиск идей для контента (каждые 6 часов)
7. **Отчеты** — сводки по лидам в 9:00, 14:00, 19:00 (только лиды от "живых людей", фильтр по `author_type`)

**Используется:** `AsyncIOScheduler` из библиотеки `apscheduler`

## 12. Архитектура

### Два бота с раздельными Dispatchers

1. **main_bot (АНТОН)** — консультант по перепланировкам
   - Обработчики: `/start`, квиз, диалог, админ-команды
   - Интеграция с LeadHunter и ScoutParser

2. **content_bot (ДОМ ГРАНД)** — контент и посты
   - Обработчики: управление контентом, публикация

### Единый источник истины

- Bot и Dispatcher создаются только в `main.py`
- Остальные модули получают бота через `utils.bot_config.get_main_bot()` или из контекста (`message.bot`, `callback.bot`)

### Неубивайка

- Lock-файл `bot.lock` для предотвращения запуска нескольких экземпляров
- Один процесс на инстанс

## 16. Логирование

**Уровни логирования:**
- `INFO` — основные события (запуск задач, обнаружение лидов, публикация постов)
- `WARNING` — предупреждения (fallback на резервный API, ошибки загрузки изображений)
- `ERROR` — критические ошибки (записываются в БД через DbLogHandler)
- `DEBUG` — детальная отладочная информация

**Формат:** `%(asctime)s - %(name)s - %(levelname)s - %(message)s`

**Особенности:**
- Ошибки уровня ERROR автоматически записываются в таблицу `system_logs` БД
- Watchdog логирует запуски и падения процессов
- Все логи сохраняются в `watchdog.log` для аудита
