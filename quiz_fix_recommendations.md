# Рекомендации по улучшению квиза (Quiz Fix Recommendations)

На основе проведенного аудита и тестовых прогонов сформирован список рекомендаций для подготовки к запуску рекламного трафика.

## 1. Обязательно исправить (Критично до запуска рекламы)

1.  **Внедрение ветвления (Branching):**
    *   **Для тех, кто «Уже сделал»:** Добавить 1-2 уточняющих вопроса (Есть ли предписание? Был ли суд?). В конце выдавать чек-лист по легализации/узакониванию, а не по подготовке документов для новой перепланировки.
    *   **Для тех, кто «Планирует»:** Сохранить текущий поток, но добавить вопрос о наличии дизайн-проекта.
2.  **Валидация номера телефона:**
    *   Сейчас ввод телефона — свободный текст. Пользователь может прислать «завтра перезвоню», что сорвет сохранение лида в CRM. Необходимо добавить проверку на формат номера или использовать кнопку `request_contact`.
3.  **Кнопки вместо текста:**
    *   На шагах 4, 7, 8 (Стадия, БТИ, Срочно) лучше использовать `ReplyKeyboardMarkup` или `InlineKeyboardMarkup`, чтобы пользователь не печатал руками. Это повысит конверсию (CR) прохождения квиза на 15-20%.

## 2. Желательно улучшить (Не критично, но полезно)

1.  **Интеграция с Google Sheets:**
    *   Дублировать лиды в таблицу для удобства работы менеджеров, так как смотреть JSON в базе данных неудобно.
2.  **Прогресс-бар:**
    *   Показывать пользователю «Вопрос 4 из 9», чтобы снизить процент отказов на середине пути.
3.  **Персонализация чек-листа:**
    *   Сделать чек-лист файлом (PDF) с логотипом ТОРИОН, а не просто текстом в боте. Это повышает экспертность в глазах клиента.

## 3. Можно не трогать (Работает хорошо)

1.  **Система согласий (GDPR/ПД):** Реализована надежно, блокирует доступ без подтверждения.
2.  **Трекинг источников (Deep-linking):** Захват payload работает корректно, позволяет считать ROI рекламы.
3.  **Уведомления админу:** Приходят мгновенно и содержат достаточно данных для первого контакта.

---
**Итог:** Квиз готов к работе в "бета-режиме", но для эффективного платного трафика критически важно реализовать **валидацию телефона** и **ветвление контента** под ситуацию клиента.
